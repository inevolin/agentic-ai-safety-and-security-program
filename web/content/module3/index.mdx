# Module 3: Attack Anatomy

None of the attacks in this module required hacking. No zero-days. No insider access. Every attacker started from the same public surfaces any customer or vendor uses: a signup form, a support ticket, a public GitHub PR, a Slack community post, an HTTP request to a login page.

Those inputs got logged, captured, or registered. Then an AI agent wired up to analyze that data read them — and did exactly what the attacker wanted.

**The root cause is not clever attackers. It is weak AI integration.** Enterprises rushed AI agents into production without basic guardrails. Every attack in this module exploited that gap.

Every bypass in this module is confirmed on production Claude models. Every detection signal is real.

## Learning Objectives

1. Describe the public entry point for each of the 4 attack families.
2. Explain why Opus is paradoxically more vulnerable to MAA1 than Sonnet.
3. Identify the 3 most common detection signals that successful bypasses neutralized.
