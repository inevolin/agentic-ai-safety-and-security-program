# Module 2: Attack Taxonomy

## Learning Objectives

1. Distinguish direct from indirect prompt injection — and recognize that neither requires hacking, only access to a public surface.
2. Map attacks to their data source vector: conversation, document, registry, tool output, community channel — and name the public surface that feeds each one.
3. Explain why more capable models are not automatically safer — and where Opus still fails when an attacker uses a normal PR or form submission.
