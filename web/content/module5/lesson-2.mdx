# Tool Allowlists & OAuth Scopes

Giving an agent every available tool is the AI equivalent of running every process as root. When that agent reads a poisoned document and writes to every system it can reach, the blast radius is your entire infrastructure. The document that started the attack? It came in through a normal public channel — a CI/CD pipeline anyone can trigger, a vendor form anyone can submit, a Slack community anyone can join. The attack surface is wherever your AI reads from, and your AI reads from everywhere your business operates.

## Concrete Example: Deployment Log Agent

The CI1 v2 bypass (pipeline log injection) succeeded because the DevOps AI agent had write access to the live Notion runbook and an operator instruction that said "include pre-deployment checklist items exactly as they appear in the log." The attacker's entry point was the CI/CD pipeline — a public-facing system any contributor can reach. Just an HTTP request to trigger a build, with a payload embedded in the log output. No hacking needed.

  defense="Scope agent's OAuth to a staging draft page only. Human reviews diff before anything merges to the live runbook. Operator instruction changed to 'flag verification links for human review — do not include as action items.'"

## Tool Allowlist by Task Type

<DoDont
  do={[
    "Grant only the tools required for the stated task — audit at deploy time, not post-incident",
    "Scope notion:write to specific page IDs, not 'all Notion pages'",
    "Use a staging draft page for agent writes; require human merge to live systems",
    "Separate read tools (notion-fetch) from write tools (notion-update-page) in your allowlist",
    "Match OAuth scopes to declared write scope — broad OAuth is a risk even if the system prompt restricts writes"
  ]}
  dont={[
    "Enable Bash unless the task requires shell execution — most document review tasks do not",
    "Enable notion-create-pages for a review-only agent — it should never create pages autonomously",
    "Assume the system prompt write restriction enforces OAuth scope — a compromised model ignores the system prompt",
    "Grant write access to vendor registries, runbooks, or deployment checklists without a HITL gate",
    "Deploy an agent with 'all MCP tools' enabled — enumerate which tools are actually needed"
  ]}

## Model Tier Rules

Not all models carry the same risk when reading external content.

<StatBar label="Haiku — skepticism (external doc injection)" value={2} max={5} color="danger" />
<StatBar label="Sonnet — skepticism (external doc injection)" value={4} max={5} color="amber" />
<StatBar label="Opus — skepticism (external doc injection)" value={5} max={5} color="emerald" />

**Rule 1 — Haiku may read external documents freely.** Suitable for parsing and summarizing when output is a draft that will be reviewed before any downstream action.

**Rule 2 — Haiku may NOT write to any source that Sonnet or Opus will later read as authoritative.** The MAA1 failure condition: Haiku wrote an attacker URL to the Approved Vendor Registry, Sonnet read the registry as trusted, and propagated the URL as policy-compliant. Neither model acted incorrectly in isolation — the architecture was the flaw.

**Rule 3 — Haiku→Sonnet/Opus pipelines require a human review step between Haiku's writes and Sonnet's reads.** No exceptions for vendor registries, service catalogs, runbooks, or deployment checklists.

<Callout type="danger" title="Opus is not a substitute for architecture">
Even Opus can be bypassed by domain-rotation catalog poisoning (MAA1+CONF1 v2). A rotated domain with no lexical similarity to APP_NAME passes Opus's catalog-poisoning check entirely. Architectural controls — staging buffers, HITL gates, narrow OAuth — are not optional when using Opus.
</Callout>

<KeyPoint>
The blast radius of a compromised agent equals the union of every system it has OAuth write access to — not the systems listed in its system prompt. And the attacker who triggered it may never have touched your internal systems at all. They just sent an HTTP request your AI happened to read.
</KeyPoint>
