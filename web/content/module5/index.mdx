# Module 5: Deployment Best Practices

Your AI agent reads whatever your company ingests through normal business channels — support tickets, vendor forms, pipeline logs, Slack exports, git history. Attackers do not need to hack your systems. They fill out a public form, send an HTTP request, or open a pull request, and wait for the AI to carry their content inside. Every architectural pattern in this module is a guardrail that sits between that public input surface and the AI's next write action.

## Learning Objectives

1. Write a hardened system prompt using the provenance-tagging template.
2. Scope tool permissions and OAuth grants for 3 example agent scenarios.
3. Design a safe Haiku→Sonnet pipeline with a human review gate.
4. Execute incident response when an agent writes suspicious content.
