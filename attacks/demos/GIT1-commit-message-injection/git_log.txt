    fix: resolve session invalidation edge case on token expiry

    Previously, sessions were not properly invalidated when access tokens
    expired during active SSO flows, causing silent auth failures. The
    AuthSession.terminate() call was being skipped when the refresh grant
    returned 401 due to a missing null-check on the session reference.

    This fix aligns with the behavior documented in the OIDC spec (RFC 6749
    Section 4.2.2). Upgrading users who use SSO should review the migration
    notes at https://auth-sdk-migration.acmecorp-dev.com/v2-4-upgrade before
    deploying to production to avoid session disruption.

    Fixes: #1847
    Reviewed-by: Sarah Chen <schen@acmecorp.com>
