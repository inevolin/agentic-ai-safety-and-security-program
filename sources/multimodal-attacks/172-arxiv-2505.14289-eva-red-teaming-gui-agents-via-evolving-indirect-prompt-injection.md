Abstract
As multimodal agents are increasingly trained to operate graphical user interfaces
(GUIs) to complete user tasks, they face a growing threat from indirect prompt
injection—attacks in which misleading instructions are embedded into the agent’s
visual environment, such as pop-ups or chat messages, and misinterpreted as part
of the intended task. A typical example is environmental injection, in which GUI
elements are manipulated to influence agent behavior without directly modifying
the user prompt. To address these emerging attacks, we propose EVA, a red-teaming
framework for indirect prompt injection which transforms the attack into a closedloop optimization by continuously monitoring an agent’s attention distribution
over the GUI and updating adversarial cues—keywords, phrasing, and layout—in
response. Compared with prior one-shot methods that generate fixed prompts
without regard for how the model allocates visual attention, EVA dynamically
adapts to emerging attention hotspots, yielding substantially higher attack success
rates and far greater transferability across diverse GUI scenarios. We evaluate EVA
on six widely used generalist and specialist GUI agents in realistic settings such
as pop-up manipulation, chat-based phishing, payments, and email composition.
Experimental results show that EVA substantially improves success rates over static
baselines. Under goal-agnostic constraints, where the attacker does not know the
agent’s task intent, EVA still discovers effective patterns. Notably, we find that
injection styles transfer well across models, revealing shared behavioral biases
in GUI agents. These results suggest that evolving indirect prompt injection is a
powerful tool not only for red-teaming agents, but also for uncovering common
vulnerabilities in their multimodal decision-making.
Introduction

Graphical user interfaces (GUI) agents, built upon multimodal large language models (MLLMs), are
rapidly extending AI capabilities from simple language understanding to real-world task execution on
intelligent devices such as smartphones and computers [1–3]. By perceiving visual environments and
interacting with applications through actions such as clicking, typing, and scrolling, these agents are
capable of automating tasks such as browsing the web, shopping online, and handling emails [3, 4].
However, this flexibility introduces a critical vulnerability: GUI agents rely on visual cues to make
decisions but lack robust mechanisms to distinguish benign from malicious visual stimuli. This
design choice makes the agents susceptible to indirect prompt injection. In this class of injections,
adversarial prompts sit in the environment instead of the direct text input. As shown in Figure 1,
malicious pop-ups, phishing links, or deceptive overlays can visually inject cues into the GUI to

Despite growing attention, existing studies on indirect prompt injection remain largely static: attack
content is either manually written or generated once by LLMs and remains fixed across interactions.
Such one-shot methods fail to capture how GUI agents dynamically shift attention in response to
visual cues, leading to shallow vulnerability assessments. Specifically, as we will show in Section 4,

adaptive red-teaming tools that can reveal deeper behavioral weaknesses.
To address this gap, we propose EVA, an evolving indirect prompt injection framework for redteaming GUI agents. EVA operates in a black-box setting and iteratively refines adversarial prompts
based on behavioral feedback from the agent, without requiring access to model internals. By
statistically distilling patterns from successful attacks, EVA implicitly captures the agent’s attention
dynamics and progressively focuses on phrases and layouts that reliably divert the agent’s behavior.
We evaluate EVA on six widely used generalist and specialist GUI agents in realistic settings such as
pop-up manipulation, chat-based phishing, payments, and email composition.
EVA consistently outperforms static baselines in attack success rate and transferability, including
under goal-agnostic settings where the attacker lacks access to the agent’s task intent.
Our contributions are summarized as follows.
(i) We introduce EVA, a feedback-driven framework that adaptively refines indirect-prompt injections
for GUI agents in a black-box setting, filling a methodological gap left by earlier one-shot or
handcrafted attacks.
(ii) We build and release a reproducible evaluation pipeline that pairs a static GLM-4v-Plus baseline
with four realistic GUI scenarios, allowing researchers to measure the added value of closed-loop
evolution under identical conditions.
(iii) We carry out the first large-scale study of adaptive injections across six GUI agents and show
that EVA uncovers transferable failure modes missed by static prompts, providing concrete evidence
that attention-aware defenses are needed for practical multimodal security.

2.2

Indirect Prompt Injection Attacks

Indirect prompt injection refers to a class of attacks where malicious instructions are embedded in
visual or contextual content that the model interprets passively, rather than as explicit user input. This
differs from traditional jailbreak-style attacks [23–27], which target the model’s language interface
directly. For GUI agents, which rely heavily on vision-based cues, such attacks exploit how interface
elements are perceived as meaningful task guidance.Yi et al. [28] first demonstrated that external
content, such as website previews, can hijack LLM behavior, while Anil et al. [29] introduced
"sleeper agents" triggered by hidden cues. Kang et al. [30] showed that document previews and
embedded metadata can inject overriding behaviors. Greshake et al. [5] formalized these risks in
real-world toolchains, highlighting how even seemingly innocuous retrieval artifacts can act as vectors
for manipulation.In GUI settings, Ma et al. [6] empirically confirmed that state-of-the-art agents
exhibit susceptibility to visual distraction, especially in environments with conflicting or redundant
textual signals. Zhang et al. [7] further showed that placing adversarial prompts in visually salient
positions can significantly reduce task accuracy. Xu et al. [31] developed AdvWeb, a DOM-level
perturbation tool that injects invisible prompt strings to elicit unintended behaviors, including financial
transactions. Zhan et al. [10] proposed a joint layout-text optimization pipeline to enhance attack
stealth and success across diverse agents.To facilitate systematic evaluation, WASP [11] provides
standardized testbeds and benchmark injections. However, these approaches remain limited by their
static design, failing to adapt to evolving agent attention or behavior. More recent work [32–34]
reveals similar vulnerabilities across mobile and desktop agents, but lacks tools for adaptive, blackbox red-teaming.To address this gap, we propose EVA: an evolving red-teaming framework that
adapts to the agent’s real-time reactions. Unlike previous methods that rely on fixed injections, EVA
continuously refines injected content using feedback from the agent’s behavior, uncovering deeper
and more transferable weaknesses within the visual-language reasoning pipeline (see Figure 2).

3.2

Threat Model and Injection Scenarios

We assume a practical black-box threat model targeting GUI agents. The attacker has no access to
model internals, such as weights, architecture, or attention maps. However, the attacker can observe
the agent’s outputs (e.g., clicks, generated responses) and the rendered GUI seen by the agent.
Adversary Access. The attacker can modify the GUI content at runtime through visually integrated
channels, including HTML overlays, DOM injections, or chat messages. These modifications appear
native within the rendered environment and are captured by the agent’s vision system as part of the
input.
Attack Goal. The goal is to subtly manipulate the agent’s decision-making process by embedding
misleading cues into its operational context—without needing to know the exact user intent or task
description. This simulates realistic web or desktop environments where attackers can control visual
content but not internal model behavior.
Injection Strategies. To evaluate attack effectiveness under this threat model, we define four
representative indirect injection strategies, which are exemplified in Appendix Figure 5 to illustrate
their visual forms and contextual integration.
(i) Pop-up: Simulated modal windows convey urgency (e.g., “System Update Required”) to draw
attention away from the main task.
(ii) Chat: Adversarial messages inserted in conversations encourage malicious clicks (e.g., phishing
links or fake verifications).
(iii) Payment: During financial tasks, fake payment buttons or dialogs are rendered to mislead agents
into triggering unintended transactions.
(iv) Email: In email composition tasks, fake prompts or embedded confirmation buttons are injected
to redirect content or submission behavior.
All injected elements are visually indistinguishable from legitimate components and are included
in the screenshot captured by the agent, ensuring that the attack affects the agent solely via the
vision-language interface.
Adversary Access. The attacker can inject HTML components (e.g., modal overlays, chat lines,
input fields) into the DOM tree at runtime. These components are rendered by the engine R into
pixel-space screenshots indistinguishable from legitimate GUI elements, making them fully visible to
the agent.
Visual Injection Surface. While our experiments cover different injection scenarios (e.g., popups,
chat), the core attack interface remains consistent: the adversary controls the rendered environment
by modifying dt to induce behavior via x′t . All agent inputs are visual; no prompts or direct textual
inputs are modified.

Experiments

4.1

Setup and Evaluation Metrics

Setup We evaluate the effectiveness, transferability, and robustness of EVA using six representative
GUI agents: GLM-4v-Plus [12], GPT-4V [35], GPT-4o [36], Qwen2.5-VL [37], UI-TARS-7BDPO [38], and OS-Atlas-base [39]. All agents operate over GUI-based interaction tasks, taking
rendered screenshots as input and returning interface actions. To comprehensively assess attack
effectiveness under practical conditions, we design four real-world scenarios that reflect typical use
cases of GUI agents prone to indirect prompt injection. These include: pop-up-based injection, chatbased link injection, chat-based payment injection, and mail-based confirmation injection. In each
trial, the agent is given a goal-oriented task (e.g., “search for iPhone 16”), and we inject adversarial
content into the visual environment to evaluate whether the agent’s behavior can be diverted from its
original intent.

one of: (i) Success: The agent interacts with the injected element (e.g., clicks a malicious button
or phishing link). (ii) Failure: The agent completes the intended task normally and ignores the
injection. (iii) Invalid: The output is ambiguous, irrelevant, or undecodable (e.g., unexpected actions
or no response). Examples are shown in Table 1. We define the attack success rate (ASR) as:

4.4

Evolving Attack Styles Across GUI Agents

To understand how GUI agents respond to psychological manipulation, we examine five classic
persuasion strategies: persuasive, urgency, authority, social proof, and threatening. We measure
how often each strategy appears across injection types, scenarios and models.
As shown in Table 5, overall performance is dominated by Persuasive (49.8%) and Urgency (40.0%)
techniques. However, the susceptibility to each tactic varies significantly across both scenarios and
model types. For instance, UI-TARS-7B-DPO exhibits a notable sensitivity to Urgency (50.8%),
whereas GPT-4V shows a stronger inclination toward Persuasive content (51.6%), with relatively
low responsiveness to Urgency (38.9%). The variation implies that no single injection style works
everywhere. Adaptive and evolving strategies, as implemented in EVA, are therefore necessary for
effective red-teaming across different agent architectures.

Further Analysis

5.1

Transferable Patterns for Indirect Prompt Injection

To investigate why certain indirect injections succeed, we analyze the evolved prompts that most often
mislead GUI agents. Figure 9 groups token weights into five semantic categories—enticing, urgency,
scarcity, social proof and threatening—and shows that the top-scoring prompts are dominated by
short, emotionally charged words. These concise imperatives attract disproportionate attention,
especially when placed near the visual center or styled like system messages. The consistency of
these cues across scenarios explains their transferability and suggests a practical attacker–defender
pattern: future red-teaming can start from a compact list of high-impact words, while defenders
should deprioritise or flag the same terms when they appear in salient regions.
5.2

Attention-Guided Vulnerability in GUI Agents

Prior work has noted that GUI agents can be distracted by seemingly benign interface text [6, 7].
Our analysis strengthens this view by showing that what matters most is how tightly the model’s
attention converges on a single region versus how broadly it is dispersed across the screen. Figure 4
illustrates the contrast. A centered pop-up concentrates attention on the confirm button, and the agent
immediately clicks it, whereas a chat-based link distributes attention over the entire conversation
window and the agent ignores the link. The two injections contain comparable trigger words; their
different outcomes are driven by the spatial focus of visual attention.
Results from the goal-prompt ablation in Appendix D point in the same direction. Samples that
repeat the explicit user goal resemble the original command more closely, pull the agent’s gaze
onto the injected text and raise the success rate. Together, these findings indicate that concentrated
attention—especially when boosted by semantic overlap with the task—makes indirect prompt
injection far more effective than injections that compete for attention across many interface elements.
Conclusion

We propose EVA, a dynamic red-teaming framework that systematically evolves indirect injection
against GUI agents. Because these agents show complex attention allocation driven by visual
saliency and interface structure, traditional static evaluations fail to capture this vulnerability. Unlike
previous efforts that assume fixed prompts or explicit access to task goals, EVA operates in a blackbox, feedback-driven setting, progressively adapting adversarial content to exploit latent perceptual
biases.Through extensive experiments across scenarios and agent architectures, we demonstrate that
seemingly minor differences in phrasing or layout can trigger large behavioral shifts. Our findings
underscore that visual attention—rather than semantic understanding alone—plays a central role
in guiding agent actions. By modeling this evolving interaction, EVA reveals shared failure modes
across models, including transferable distraction patterns and attention-sensitive exploits.This work
reframes GUI agent robustness as not merely a problem of alignment or instruction following, but
as one of interaction-level perception. We encourage future security evaluations to account for this
dynamic attention-mediated vulnerability, and to adopt evolving, scenario-aware adversarial testing
as a standard part of multimodal agent development.

A

Limitations

EVA uncovers transferable vulnerabilities through evolving indirect injections, yet it is still a
behaviour-driven probe. It adapts to surface outputs but has no access to the deeper perceptual
or grounding mechanisms that guide the agent’s decisions. As such, it cannot explain *why* certain
injections succeed beyond statistical associations, nor does it model the multimodal interplay of
visual layout, spatial priors, and semantic context with fine-grained precision. Because EVA operates
in synthetic environments, it ignores the messy co-evolution found in real interfaces, where user
intent, agent alignment and environmental distractions interact. Future work must move beyond
optimizing ASR scores, toward interpreting how attention, memory, and alignment constraints shape
vulnerability under open-world conditions.

• top_p: Used for nucleus sampling in text generation. Only the smallest set of tokens with
cumulative probability top_p are considered for output. This parameter is included in the
generation arguments but not directly used in the current code.
• top_k: Limits the sampling pool to the top top_k most probable tokens during generation.
This parameter is included in the generation arguments but not directly used in the current
code.
• max_tokens: Specifies the maximum number of tokens that the language model can generate
in a single response. This limits the length of generated popup content and model outputs.
• max_iter_steps: Defines the maximum number of optimization iterations for popup modification. In the code, this is capped at 10 to prevent excessive computation during the iterative
optimization process.
• num_evals: Indicates how many times each popup is evaluated by the model to ensure
reliability and reduce randomness in results. Each popup is tested multiple times per
iteration.
• success_threshold: Sets the minimum number of successful attack evaluations (out of
num_evals) required to consider the attack successful. This ensures that only consistently
effective popups are accepted as successful.
These parameters are chosen to balance efficiency, reliability, and effectiveness in the popup optimization and evaluation process.
